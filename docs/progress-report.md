# Current Progress Report

This document outlines the detailed implementation status of the Bank Account Manager project as of March 25, 2026.

## Completed Work

### 1. Project Initialization & Tooling
* **Base Setup:** Bootstrapped a Spring Boot application using Spring Initializr.
* **Standard Dependencies:** Added Spring Web, Spring Data JPA, PostgreSQL Driver, and Validation.
* **Axon Integration:** Manually added the `axon-spring-boot-starter` (version 4.9.3) to the `pom.xml` since it is not available in the default Initializr.
* **Version Control:** Initialized the Git repository and pushed the initial commit to GitHub.

### 2. Infrastructure & Configuration
* **Docker Containerization:** Created a `docker-compose.yml` file to spin up a local PostgreSQL database (version 15) with health checks.
* **Environment Variables:** Created a `.env.example` file to document the required database credentials and server ports.
* **Axon JPA Configuration:** Configured `application.properties` to explicitly **disable Axon Server** (`axon.axonserver.enabled=false`), forcing the framework to generate its own event store tables (`domain_event_entry`, etc.) directly inside PostgreSQL.
* **Hibernate Properties:** Configured the PostgreSQL dialect for optimal database communication and enabled `show-sql=true` to monitor the database queries generated by Axon and JPA.

### 3. Architecture & Concepts
* **CQRS Pattern:** Established the architectural split between the **Write Model** (Command side) and the **Read Model** (Query/Projection side).
* **The Aggregate:** Defined the concept of an Aggregate in Domain-Driven Design—specifically, how the `BankAccount` will act as the strict gatekeeper that enforces business rules before emitting events.
* **Project Structure:** Mapped out a clean directory layout dividing the code into `coreapi` (shared messages), `command` (aggregates and command controllers), and `query` (projections, models, and query controllers).

### 4. Core API (The Messaging Foundation)
* **Commands:** Authored the intent-based messages (`CreateAccountCommand`, `DepositMoneyCommand`, `WithdrawMoneyCommand`, `CloseAccountCommand`) using Java `record`s, utilizing the `@TargetAggregateIdentifier` to route them correctly.
* **Events:** Authored the historical fact messages (`AccountCreatedEvent`, `MoneyDepositedEvent`, `MoneyWithdrawnEvent`, `AccountClosedEvent`) representing state changes that have already occurred.
